Did you see how this chapter started on a new page? This would occur even if we chose not to use CSS styling to achieve this effect. The reason is that each chapter is split into its own HTML files, and almost every eReading device will automatically start a new HTML file on a new page. It is wise to split up your HTML files this way. If your readers try to open one big monstrous HTML file that contains the entire content of your book, it will be very slow and a painful experience.

You can add images, audio, video, fonts, and some other things to your EPUB package and utilize them throughout your HTML. You just have to make sure that they are declared in the Manifest Section of the content.opf file. Below is a picture:
