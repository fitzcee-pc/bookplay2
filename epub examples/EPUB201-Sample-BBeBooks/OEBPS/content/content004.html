Chapter 3 – Caution

You have to be very careful when building your EPUB package, because a lot can go wrong. Make sure that everything inside the EPUB package (every HTML, CSS, and media file) is declared in the manifest section with the appropriate mimetype. Another issue is that when you split your source HTML file into many smaller ones, make sure that your relative links in your HTML files to your CSS file and images match based on the sub-directory structure of the EPUB package.

For example, the correct src attribute in our HTML would be ../images/someimage.jpg, which ensures that the relative path is accurate, since we have all of our content in a folder called content, and all our images in a folder called images.

You should always validate your EPUB file for errors after you package. You can either download the EpubCheck command-line program here or use the IDPF’s free online validator service. This can catch a lot of errors that may not be obvious.
